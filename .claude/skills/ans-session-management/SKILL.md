# ans-session-management

## File Map

| File | Purpose |
|---|---|
| `server/session_manager.py` | `SessionManager` class and `AgentSession` dataclass. Registry of all connected MCP client sessions. Wraps `CallManager` for voice ops, integrates `VoicePool` and `Switchboard`. |
| `server/voice_pool.py` | `VoicePool` class. Assigns distinct TTS voices to concurrent sessions so users can tell agents apart by sound. |
| `server/voice_profile.py` | `VoiceProfileRegistry` and `VoiceProfile`. Manages preset speakers (Qwen3-TTS built-ins) and cloned voice profiles (`voices/<name>/profile.json`). |
| `server/switchboard.py` | `Switchboard`, `MessageQueue`, `QueuedMessage`. Per-session message queues for agent-to-user and user-to-agent delivery. |
| `server/router.py` | `IntentRouter` and `RouterIntent`. LLM-powered intent classification to route user speech to the correct session. Uses `thefuzz` for fuzzy name matching. |
| `server/check_messages.py` | `check_messages()` function. MCP tool that retrieves pending user-to-agent messages from the switchboard. |
| `server/session_browser.py` | `SessionBrowser` and `SessionMetadata`. Reads previous Claude Code / Codex session history from filesystem for `/sessions` and `/resume`. |

## Architecture Overview

```
MCP Client (HTTP)
    |
    v
SessionManager
    |--- register_session() --> VoicePool.assign_voice()
    |                       --> Switchboard.register_session()
    |
    |--- initiate_call() ----> CallManager (voice ops)
    |    continue_call()       resolves voice via VoicePool
    |    speak_to_user()
    |    end_call()
    |
    |--- list_active_sessions()  (includes switchboard queue counts)
    |
    v
Switchboard                    IntentRouter
    |--- enqueue_agent_message()    |--- classify(transcript, sessions)
    |--- enqueue_user_message()     |    uses LLM + fuzzy match
    |--- get_pending_announcements()
    |--- deliver_next_message()
    v
check_messages() MCP tool <-- reads from Switchboard
```

## Key Patterns

### Session ID Hierarchy

Two distinct ID layers exist:
- **MCP session ID**: Assigned by the Streamable HTTP transport. Identifies the HTTP connection.
- **Agent session ID**: UUID generated by `SessionManager.register_session()`. Used everywhere internally.

`SessionManager._mcp_to_session` maps between them. The `get_session_by_mcp_id()` method resolves MCP IDs to agent sessions.

### Voice Resolution (Single vs Multi)

Voice assignment follows a dual-mode rule:
- **Single session active**: Always uses the default voice from config. No pool assignment matters.
- **Multiple sessions active**: Each session uses its pool-assigned voice.

`VoicePool.resolve_voice()` implements this: if `len(assignments) <= 1`, return `default_voice`; otherwise return the assigned voice. `SessionManager.resolve_voice()` delegates to the pool.

### Voice Pool Assignment Priority

1. Already assigned to this session -> return existing
2. Explicit `requested_voice` available (not assigned, not system voice) -> assign it
3. Next unassigned voice from the ordered pool list -> assign it
4. All pool voices exhausted -> reuse least-common voice with warning

System voice is never assigned to agent sessions (reserved for switchboard announcements).

Note: The `/voices` Discord slash command can change the default voice at runtime, which updates `VoicePool._default_voice`.

### Switchboard Message Flow

Two directions:
- **agent_to_user**: Agent enqueues a message when it cannot speak (another session has the channel). System Voice announces these via `get_pending_announcements()`.
- **user_to_agent**: User sends a cold-call to a busy agent. Agent retrieves these via `check_messages` MCP tool.

Messages are `QueuedMessage` dataclasses with `delivered` flag. Queues have a max depth (`max_queue_depth`, default 20) to prevent unbounded growth.

### Router Intent Classification

Four intents:
- `reply_current` -- default, reply to whoever just spoke
- `route_to_session` -- direct reply to a named session
- `cold_call` -- async message to a named session
- `navigation` -- switchboard control (next/skip/list)

Fuzzy matching via `thefuzz.process.extractOne` with `token_sort_ratio` scorer, threshold 75. Falls back to `reply_current` on any error, timeout, or unmatched session name.

### Session Browser

`SessionBrowser` reads previous CLI session history from the filesystem. Key methods:

- `list_claude_sessions(directory)` -- list sessions for a specific project
- `list_codex_sessions(directory=None)` -- list Codex sessions, optionally filtered
- `list_recent(n=10, cli_filter=None)` -- merge and sort all sessions by recency
- `find_session(session_id)` -- look up a session by ID, returns `SessionMetadata` with cli type, directory, summary, timestamp (or `None` if not found)
- `detect_cli(session_id)` -- returns `"claude"`, `"codex"`, or `"unknown"`

`find_session()` is used by `/resume` to resolve both the CLI type and original working directory.

Filesystem paths:
- Claude Code: `~/.claude/projects/<encoded-path>/sessions-index.json`
  - Path encoding: `/home/joe/myproject` becomes `-home-joe-myproject`
- Codex: `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`
  - First line is `{"type": "session_meta", "payload": {...}}`

### Name Collision Avoidance

`SessionManager._unique_name()` uses a `Counter` to track base names. First use of "myproject" stays "myproject". Second becomes "myproject-2". Counter decrements on unregister.

## Config Keys

| Config Key | Default | Used By |
|---|---|---|
| `voice_pool` | `["Ryan", "Aiden", "Vivian", "Serena", "Dylan", "Eric"]` | `VoicePool` |
| `tts.default_voice` | `"Ryan"` | `VoicePool` (single-session voice) |
| `system_voice` | `""` | `VoicePool` (reserved, never assigned) |
| `max_queue_depth` | `20` | `Switchboard` (per-session queue limit) |
| `tts.voices_dir` | `"voices"` | `VoiceProfileRegistry` (clone scan dir) |
| `tts.elevenlabs.voices` | `{}` | ElevenLabs voice name->ID mapping |
| `router.enabled` | `false` | `IntentRouter` (skip LLM if disabled) |
| `router.model` | `""` (falls back to `llm.model`) | `IntentRouter` |
| `router.timeout_ms` | `500` | `IntentRouter` |
| `llm.backend` | `"openrouter"` | `IntentRouter` (shared LLM config) |
